## 参考案例库规划

后续建议创建一个专门仓库：

```text
ref-reference-library/
  ppt-cases/
  website-cases/
  game-cases/
  design-styles/
  business-cases/
  readme-examples/
  interaction-patterns/
  links.md
```

这个仓库只放“参考资料”和“案例模板”，不放最终作品。以后生成 PPT、网站、游戏或工具前，可以先读取这里的案例，再开始创作。

## 维护规则

- 每个仓库至少保留 `README.md`。
- README 写清楚：项目类型、用途、技术栈、在线地址、截图、后续计划。
- GitHub Topics 使用英文标签，方便搜索和筛选。
- 成熟作品单独建仓库；零散素材可以先进入合集仓库。
- 参考案例和模板单独放到 Reference / Templates，不混进正式作品。
